Updated the sidebar to expose all headwords without the old 2,000-entry cap.

The key change is in src/main.c: the sidebar now uses a virtual GtkListView backed by a custom GListModel over the mmap’d FlatIndex. That means the full headword count is available for scrolling, but GTK only creates visible row widgets, so RAM and CPU stay low even for large dictionaries.

Search still uses the existing filtered index array, but the empty-search sidebar now streams directly from the flat index instead of prebuilding/appending thousands of rows.
